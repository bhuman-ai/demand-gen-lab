# Feature: brand-gpt-operator-chat-intelligence

## Request
Improve the brand GPT / Operator chat because it feels dumb: when the user asks a simple brand-specific question like 'any replies?' it should answer with actual reply/inbox data and clear next steps, not generic sender warmup boilerplate. Also reduce confusing GPT vs Operator labeling if needed.
## Autonomy Mode
holistic_autopilot
## Target Users
Founder/operator managing outbound brands
## Optimization Target
decision speed and trustworthy brand-specific answers
## Hard Constraints
- Do not add hardcoded campaign strategy logic as a substitute for model reasoning.
- Factual account answers must be grounded in current brand context, tool results, or explicit planner failure.

## Scope
Route Brand GPT through the shared LLM planner path, make planner failures visible, and clean up confusing user-facing labels/starter behavior.
## Touched Surfaces
- Brand page right-side GPT/Operator chat panel
- /api/operator/chat
- operator runtime data grounding
## Success Moment
The operator asks a normal brand-specific question and sees an answer generated by the planner against current brand state, not a generic fallback status template.

## Failure Policy
If the planner is unavailable, show that explicitly and do not imply account inspection or changes happened.

## Primary Action
Ask a brand-specific question and get a direct, grounded answer first.
## Primary Risk
The AI may answer from stale or incomplete data, or substitute generic advice for the actual requested fact.
## Information Budget
The panel should show the direct answer, the key source counts or recent items, and at most one next action. Extra audit detail stays in technical details or follow-up prompts.
## View Model Contract
Primary user: Founder/operator managing outbound brands
Current decision: What is happening for this brand right now?
Why now: The operator needs to trust GPT before allowing it to run campaigns autonomously.
Next action: Ask a plain-language question or approve one concrete action.
Top risk: The panel sounds smart but does not use the actual brand data.
## Concept Options
Option A: Rename-only cleanup. Keep current Operator workflow but change labels and starter prompts. Low risk, does not fix dumb answers., Option B: Answer-first Brand GPT. Keep the side panel, rename the user-facing label to Brand GPT, detect common factual questions like replies/inbox/results, answer from live brand data, then optionally suggest the next action. ## Concept Winner

Generic operational guidance only appears after the factual answer, and labels should consistently say Brand GPT while internal implementation can remain operator-runtime.
