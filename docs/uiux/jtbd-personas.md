# JTBD & Personas

## Primary Jobs to be Done
Founder/operator wants to give the brand website and target customers to an AI operator, then trust it to source leads, test messaging, manage deliverability, send outbound, watch replies, learn, and ask for help only when useful.
## Primary Personas
- Founder/operator: wants growth moving without babysitting tooling.
- Growth operator: inspects evidence, fixes blockers, and reviews replies when the AI escalates.
- Technical/admin user: connects providers, sender infrastructure, billing, credentials, and diagnostic tools.
## Baseline Anxiety / Stress States
High. The user is worried the agent is fake, stuck, too hardcoded, spending money, hurting deliverability, or silently not moving. UI should answer: is it working, what did it do, why, what is blocked, and what happens next?
## Permissions & Roles
- Founder/operator sees Brand GPT first and should not need to understand internal tool wiring to know what is happening.
- Growth operator needs evidence, reply review, deliverability status, and manual intervention controls, but these should sit one layer behind the default agent view.
- Technical/admin user needs provider setup, credentials, diagnostics, and system logic. These belong in secondary System surfaces, not the main work nav.
